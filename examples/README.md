# Examples

Sample inputs and expected Markdown outputs for each supported format.

## Directory Structure

| Folder | Format | Contents |
|--------|--------|----------|
| `docx/` | Word .docx | Expected output for a typical PM document |
| `pdf/` | PDF | Expected output for a PDF with headings, tables, and lists |
| `confluence/` | Confluence HTML | Sample HTML export and expected Markdown output |

## How to Use

1. Compare your conversion output against the expected output files
2. Use these as reference for the quality bar we're targeting
3. Binary source files (.docx, .pdf) will be added manually

## Quality Standard

Every example output demonstrates:
- Single H1 (document title)
- Clean heading hierarchy (no skipped levels)
- Consistent list markers (`-` for unordered, `1.` for ordered)
- Properly formatted pipe tables
- No smart quotes or phantom formatting
- Straight quotes throughout
